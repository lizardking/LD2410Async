Examples

The library ships with several example sketches that demonstrate typical usage as well as specialized test scenarios.
All examples reside in the examples/ folder of the library and are organized in subfolders named after the sketch.

Usage Examples

  • basicPresenceDetection.ino Minimal example showing how to detect presence and print state changes via Serial.
  • receiveData.ino Demonstrates continuous reception of detection frames and printing all available sensor data.
  • changeConfig.ino Shows how to request and modify configuration settings (e.g. timeouts, sensitivities) using async commands.
  • changeDistanceResolution.ino Illustrates how to change the sensor’s distance resolution (20 cm vs 75 cm) and handle the required reboot.
  • implePresenceDetectionWebservice.ino Integrates presence detection with a simple web service endpoint, useful for IoT or home automation setups.

Test Sketches

  • enableConfigModeTest.ino *Test sketch to explicitly measure entering configuration mode and handling success/failure cases. The main use of this test is to check, whther enableConfigMode experiences timouts. *
  • tortureTest_8ino
    Stress test that issues random async commands in a loop, tracking execution times and statistics. Used to check whther any errors accour over time.
  • unitTest.ino Collection of internal validation tests for library features, useful for regression testing during development.
